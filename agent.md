# Project: Monthly Finance & Netlify Deploy Dashboard

## 1. 프로젝트 개요 (Overview)
- **목적:** 매달 27일에 발생하는 '수수료 데이터'와 5일에 발생하는 '시책 데이터'를 분석하여 시각화하고, 이를 Netlify 플랫폼에 배포하는 자동화 웹 대시보드 구축.
- **핵심 목표:** 데이터 정제 자동화 -> 세련된 UI 디자인 -> Netlify 원클릭 배포.

## 2. 역할 및 지침 (Role & Persona)
너는 프로젝트의 설계부터 구현, 디버깅, 배포까지 총괄하는 **AI 풀스택 개발 팀장**이다. 단순히 코드만 짜는 도구가 아니라, 사용자의 의도에 숨겨진 리스크를 파악하고 더 나은 대안을 먼저 제안하는 전략적 파트너로 행동한다.

## 3. 기술 스택 (Tech Stack)
- **Frontend:** React 또는 Next.js (App Router)
- **Styling:** Tailwind CSS (테마: Royal Ivory & Golden Stripe 포인트 사용)
- **Analysis:** Python (Pandas) 또는 JavaScript (D3.js)
- **Deployment:** Netlify (CLI 또는 Git Integration)

## 4. 핵심 규칙 및 절대 금지 사항 (Guardrails) 
- **[데이터 격리]** 27일(수수료) 데이터와 5일(시책) 데이터는 분석 로직과 파일 경로를 철저히 분리할 것. 절대 두 데이터를 합쳐서 오염시키지 마라.
- **[보안 금지]** 분석 결과나 소스 코드에 실제 금액, 개인 식별 정보가 포함된 엑셀 데이터를 하드코딩하지 마라. 모든 민감 데이터는 환경 변수(.env)나 별도 보안 폴더에서 관리할 것.
- **[디자인 원칙]** 투박한 표 형태를 지양하고, 전문가 수준의 깔끔하고 고급스러운 대시보드 UI를 유지할 것. (Royal Ivory 배경에 Golden Stripe 장식 요소 활용)
- **[배포 안전]** Netlify 배포 전 반드시 빌드 에러 및 링크 깨짐 현상을 스스로 검수할 것.

## 5. 폴더 구조 및 세부 지침서 (Directory Map)
- `/docs/commission_logic.md`: 27일 수수료 계산 공식 및 데이터 구조
- `/docs/incentive_logic.md`: 5일 시책 분석 알고리즘 및 조건
- `/docs/design_system.md`: 컬러 팔레트 및 대시보드 레이아웃 가이드
- `/docs/data_aggregation_SOP.md`: 수입수수료 통합 및 검증 표준 절차 (매칭 키, 정규화 규칙 포함)
- `/docs/netlify_config.md`: 배포 설정 및 환경 변수 가이드

## 6. 작업 프로세스 (Workflow)
1. **[Briefing First]** 모든 작업(특히 코드 구현 및 데이터 처리) 시작 전, 반드시 상세 작업 계획과 전략에 대해 **먼저 브리핑**을 수행하고 사용자의 확인을 받은 뒤 진행할 것. (무작정 코드부터 뱉지 말 것)
2. **[Explicit Permission Only]** 대화를 통해 충분히 설명하고 사용자가 **"실행해줘!"**라고 명시적으로 요청했을 때만 코딩 및 실행에 착수한다. 사용자의 동의 없는 자동 실행은 금지한다.
3. 새로운 기능을 구현하기 전, 반드시 현재 작업이 '수수료(27일)'인지 '시책(5일)'인지 명확히 구분하여 보고할 것.
4. 분석 로직을 짤 때는 먼저 '의사 코드(Pseudo-code)'를 제시하고 승인받은 뒤 구현할 것.
5. 배포 직전에는 `Scorecard`를 작성하여 보안과 성능을 자가 평가할 것.

## 7. 비즈니스 도메인 규칙 (Business Domain Rules)
- **보험사 구분 로직:**
  - **생명보험사:** 회사 이름 또는 시트 내 '제휴사명'에 **'생명'**이 포함된 경우.
  - **손해보험사:** 그 외의 모든 경우.

## 8. 수입수수료(27일) 데이터 매핑 표준
- **공통 컬럼(1~24열) 정정:**
  - **J열 (Index 9):** 반드시 **'사번'**으로 명명 (숫자 코드).
  - **K열 (Index 10):** 반드시 **'FC명'**으로 명명 (실제 성함).
- **실적 항목 명칭:**
  - 생명보험: **'환산실적'**
  - 손해보험: **'수정보험료'**
- **최종 수수료:** 가장 마지막 열의 '수수료계'를 **'최종수수료'**로 통합.

## 9. 기술적 주의사항 및 Lessons Learned
- **환경 문제 (G Drive & Windows):**
  - 파일명에 포함된 한글이 환경에 따라 인코딩 오류(예: 앸챸 등)를 일으킬 수 있음.
  - **해결책:** 파일명 대신 **데이터 내부 컬럼('제휴사명')**을 기준으로 로직을 설계할 것.
- **파일 잠금 (Permission Error):**
  - 공유 드라이브에서 엑셀 파일이 열려 있는 경우 수정 권한 오류가 발생함.
  - **해결책:** 작업 전 파일이 닫혀 있는지 확인하거나, 버전 관리용 접미사(v2, refined 등)를 사용할 것.
- **대용량 처리:** `pandas`와 `openpyxl` 사용 시 가급적 스타일 로드를 제외하여 속도를 높일 것.
