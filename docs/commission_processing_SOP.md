# [원샷] 보험 수입수수료 산출 및 매핑 작업 매뉴얼 (SOP)

본 문서는 매월 발생하는 보험사별 수수료 원본 데이터를 통합하고, 사용자 지정 목표 수치에 맞춰 최종 정산 파일을 만드는 **전 자동화 프로세스**를 정의합니다.

## 1. 작업 준비 및 실행 환경
1.  **입력 데이터:**
    *   **보험사 원본:** `(사업부)수입수수료산출_*.xlsx` 형식의 모든 파일 (개수 무관)
    *   **이파트너:** `*이파트너*.xlsx` 형식의 정산 파일 (1개)
2.  **사용자 필수 입력 사항:**
    *   **작업 경로:** 파일들이 위치한 폴더 경로
    *   **생명보험 목표 수치:** 최종 결과물의 생명보험 지사수수료 합계 (예: 79,913,800)
    *   **손해보험 목표 수치:** 최종 결과물의 손해보험 지사수수료 합계 (예: 16,649,425)

## 2. 핵심 작업 로직 (AI 수행 지침)

### [단계 1] 지능형 마스터 통합 (데이터 세척)
*   **파일 탐색:** 폴더 내 `(사업부)`로 시작하는 모든 파일을 읽어옴 (통합/최종본은 제외).
*   **컬럼 규격 강제 세척:**
    *   **10번 열:** 명칭을 `사번`으로 변경.
    *   **11번 열:** 명칭을 `FC명`으로 강제 지정 (원본의 `FC명.1` 등 오염 데이터 방지).
    *   **25번 열:** `환산`(생명) 또는 `수정보험료`(손해)로 지정.
    *   **26번 열:** 각 원본 파일의 **절대적 마지막 열**을 가져와서 `수수료계`로 명명.
*   **필터링:** `수수료계`가 **0원인 행은 무조건 제외**하고 `생명보험`, `손해보험` 시트로 통합.

### [단계 2] 1:1 정밀 매칭 및 수치 보정
*   **3중 키 매칭:** `증권번호 + 납입회차 + 순번(seq)`을 사용하여 중복 증권번호의 금액 뻥튀기를 방지.
*   **항목 추가:** 26번 열 옆에 `FC수수료`(27번), `지사수수료`(28번), `분담금`(29번) 컬럼 생성.
*   **목표 수치 강제 보정:** 
    *   매칭되지 않은 이파트너 정산분(환수 등)을 추출하여 시트 하단에 추가.
    *   최종 합계가 사용자가 입력한 **목표 수치와 1원까지 일치**하도록 맞춤.

### [단계 3] 클린 규격 출력
*   **열 개수 고정:** 마스터(26) + 수수료(3) = **총 29개 열** 외의 모든 중복 열(계약자명, 상품명_ep 등)은 삭제.
*   **결과 파일명:** `(사업부)수입수수료산출_최종_YYYYMM.xlsx`

## 3. 오답 노트 (반복 금지 사항)
*   보험사 원본의 `FC명.1`을 그대로 두지 말 것 ➡️ 무조건 `FC명`으로 세척.
*   이파트너의 컬럼들이 옆으로 늘어나게 두지 말 것 ➡️ 지정된 29개 열만 남길 것.
*   증권번호만으로 매칭하지 말 것 ➡️ 반드시 회차와 순번을 포함하여 1:1 매칭할 것.
