# 2604 수입수수료산출 데이터 헤더 분석 결과

2604(4월분) 수입수수료 산출 엑셀 파일(총 14개)의 헤더(1행)를 분석한 결과입니다.

## 1. 공통 항목 (Overlapping Columns)
모든 보험사 파일에서 **앞부분 24개 컬럼**은 거의 동일한 순서와 명칭으로 구성되어 있습니다. 이 데이터들은 계약의 기본 정보와 조직 정보를 담고 있어 데이터 통합 시 Key로 활용 가능합니다.

- **기본 정보:** 번호, 적용년월, 제휴사명, 증권번호
- **조직 정보:** 본사, 사업단, 지사, 지점, 팀
- **설계사 정보:** FC명, FC코드 (일부 파일에서 중복 표기)
- **계약 상세:** 지급구분, 표준상태, 기초상태, 상품군, 상품코드, 상품명, 계약일자, 태아구분, 계약자, 납입방법, 납입기간, 납입회차, **보험료**

## 2. 차이 항목 (Unique/Varied Columns)
'보험료' 컬럼 이후부터는 보험사별, 상품별(생명/손해)로 수수료와 성적 산출 항목이 크게 달라집니다.

### 주요 차이점
- **성적/실적 명칭:**
  - 대부분의 손해보험: `수정보험료`
  - 생명보험 및 일부 손해: `환산성적`, `TP`, `신월정산수정P`, `CPC`
- **수수료 구성:**
  - **손해보험:** `신계약수수료/커미션`, `자동차수수료`, `자동차성과`, `일반수수료`, `일반성과` 등 (자동차/일반 보험 유무)
  - **생명보험:** `성과수수료`, `계약관리보너스`, `장기유지성과`, `파트너쉽수수료` 등 인센티브 중심
- **기타:** `Override`, `환수수수료`, `유지수수료` 등의 포함 여부와 명칭이 조금씩 상이함.

## 3. 통합 제언 (Merging Strategy)
1. **공통 영역(1~24열):** 그대로 합쳐서 하나의 Master DB 구축.
2. **변동 영역(25열 이후):**
   - 각 보험사별로 상이한 항목들을 표준 항목(예: 성적, 모집수수료, 유지수수료, 환수 등)으로 맵핑하여 통합.
   - 맵핑되지 않는 특수 항목은 별도 컬럼으로 유지하거나 '기타 수수료' 합계로 처리.
